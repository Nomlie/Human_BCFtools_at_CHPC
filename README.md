# HCC bcftools Variant Calling Pipeline
### Parallel chunked processing per chromosome group

A lightweight PBS-based workflow for joint variant calling on human whole-genome sequencing data using:

**bcftools mpileup → call → annotate → filter**

This pipeline is designed for shared HPC environments where each user is assigned a small group of chromosomes, such as `chr1`, `chr14`, and `chr21`, and processes only those chromosomes.

---

## Features

- Fixed-size genomic chunking (default: **10 Mbp**)
- Automatic skipping of completed chunks using checkpoint files
- Parallel processing of up to **3 chromosomes** within a single **24-core SMP node**
- **8 threads per chromosome**
- Special ploidy handling for **chrM** and **chrY** using haploid calling
- dbSNP annotation in the **ID** field
- Basic **QUAL ≥ 20** filtering
- Detailed per-region timing and status logs
- Easy resume after job interruption
- Final per-chromosome VCFs, compressed and indexed

---

## Directory Layout

Example output structure for a chromosome group run:

```text
group_runs/
└── chr1_chr14_chr21/                 # CHROM_GROUP_TAG
    ├── regions.tsv                   # generated by 01_make_regions.sh
    ├── master_logs/
    ├── chunks/
    │   ├── chr1/
    │   │   ├── HCC.chr1.1_10000000.bcf
    │   │   ├── HCC.chr1.1_10000000.annotated.bcf
    │   │   └── HCC.chr1.1_10000000.filtered.vcf.gz
    │   ├── chr14/
    │   └── chr21/
    ├── logs/                         # detailed per-chunk logs
    ├── checkpoints/                  # *.done stamp files
    ├── summaries/                    # timing and status TSV files per chunk
    └── final/
        ├── HCC.chr1.bcftools.annotated.filtered.vcf.gz
        ├── HCC.chr14.bcftools.annotated.filtered.vcf.gz
        └── HCC.chr21.bcftools.annotated.filtered.vcf.gz
```

## Quick Start
1. Edit the configuration file

Each user only needs to edit:

pipeline.conf

Set your assigned chromosomes, for example:

CHROM_LIST=(chr1 chr14 chr21)
2. Run the pipeline steps in order
Generate regions.tsv

This only needs to be done once for a given chromosome group.

```bash bash run_pipeline.sh setup```
Submit the main chunk-processing job

This runs the parallel variant calling workflow on a 24-core SMP node.

```bash bash run_pipeline.sh submit```

Merge chunk outputs into final chromosome-level VCFs

Run this after the chunk-processing job completes.

```bash bash run_pipeline.sh merge```

3. Check progress at any time
```bash bash run_pipeline.sh progress```

This creates a report in:

group_runs/<your_group>/progress_report_*.txt

## Available Commands
```bash bash run_pipeline.sh <command>```

Command	    Action
setup	    Submit 01_make_regions.sh to create regions.tsv
submit	    Submit 02_run_chunks.sh for parallel variant calling
progress	Submit 04_check_progress.sh to generate a progress report
merge	    Submit 03_merge_chromosomes.sh to merge chunked outputs
help	    Show usage information

## Resuming After Failure
Re-running the pipeline

In most cases, you can simply re-submit the main job:

```bash bash run_pipeline.sh submit```

Completed chunks will be detected and skipped automatically.

If some chunks failed midway
Option A: Manual cleanup

This is the safer option.

Check the logs in:

logs/chr*/

summaries/chr*/

Delete only the incomplete files for the failed region(s)

Re-submit the job

Option B: Automatic cleanup of incomplete files

```bash qsub -v AUTO_CLEAN_INCOMPLETE=1 02_run_chunks.sh```

Configuration

Main settings are stored in pipeline.conf.

Variable	Example / Default	Meaning
CHROM_LIST	(chr1 chr14 chr21)	Chromosomes assigned to the user
CHUNK_SIZE	10000000	Chunk size in base pairs
MAX_DEPTH	1000	Value passed to bcftools mpileup --max-depth
PARALLEL_CHROMS	3	Number of chromosomes processed concurrently
THREADS_PER_CHROM	8	Threads allocated per chromosome
BAM_LIST	/path/to/HCC_BAM.txt	Text file containing input BAM paths, one per line
CONFIG_TXT	/path/to/config.txt	Shared project configuration file with reference and dbSNP paths
Output Files Per Chunk

Each processed chunk generates the following files:

File	Description
HCC.<chr>.<start>_<end>.bcf	Raw multi-sample BCF produced after mpileup and call
HCC.<chr>.<start>_<end>.annotated.bcf	BCF annotated with dbSNP rsIDs
HCC.<chr>.<start>_<end>.filtered.vcf.gz	Final compressed VCF after QUAL >= 20 filtering
