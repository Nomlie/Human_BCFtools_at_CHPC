# HCC bcftools Variant Calling Pipeline
### Parallel chunked processing per chromosome group

A lightweight PBS-based workflow for joint variant calling on human whole-genome sequencing data using:

**bcftools mpileup → call → annotate → filter**

This pipeline is designed for shared HPC environments where each user is assigned a small group of chromosomes, such as `chr1`, `chr14`, and `chr21`, and processes only those chromosomes.

---

## Features

- Fixed-size genomic chunking (default: **10 Mbp**)
- Automatic skipping of completed chunks using checkpoint files
- Parallel processing of up to **3 chromosomes** within a single **24-core SMP node**
- **8 threads per chromosome**
- Special ploidy handling for **chrM** and **chrY** using haploid calling
- dbSNP annotation in the **ID** field
- Basic **QUAL ≥ 20** filtering
- Detailed per-region timing and status logs
- Easy resume after job interruption
- Final per-chromosome VCFs, compressed and indexed

---

## Overview

This mini pipeline is a lightweight, PBS-based workflow for running **joint variant calling** on a defined group of chromosomes using **bcftools** on the cluster.

It was designed for shared HPC use, where each user is assigned a small chromosome group such as:

- `chr1 chr14 chr21`
- `chr2 chr15 chr22`

The pipeline splits each chromosome into fixed-size genomic regions, processes each region chunk by chunk, and then merges the chunk-level outputs into final per-chromosome VCF files.

The main variant-calling workflow uses:

**bcftools mpileup → call → annotate → filter**

---

## What the Pipeline Does

The workflow runs in four stages:

1. **Setup**  
   Generates the `regions.tsv` file containing the genomic chunks for the chromosomes assigned to you.

2. **Chunk processing**  
   Runs the main variant-calling workflow on each chunk.

3. **Progress check**  
   Generates a report showing which chunks are complete, missing, or still in progress.

4. **Merge**  
   Merges all finished chunk-level outputs into one final VCF per chromosome.

```bash
git clone https://github.com/Nomlie/Human_BCFtools_at_CHPC

```

## What You Need to Edit

Before running the pipeline, edit:

pipeline.config

The main setting you must update is your chromosome group:

CHROM_LIST=(chr1 chr14 chr21)

Only include the chromosomes assigned to you.

The config file may also contain parameters such as:

- CHUNK_SIZE
- MAX_DEPTH
- PARALLEL_CHROMS
- THREADS_PER_CHROM
- BAM_LIST
- CONFIG_TXT

You can also edit each script to include your email address on the PBS parameters if you want to be notified when the job starts.
Do not change cluster resource settings unless you have been instructed to do so.

## How to Run the Pipeline

The pipeline is controlled through:

```bash run_pipeline.sh <command>```

**1. Setup**

Generate the regions.tsv file for your chromosome group:

```bash run_pipeline.sh setup```

This step usually only needs to be run once, unless your chromosome list or chunk size changes. It should take lesser than an hour and produce regions.tsv file

**2. Submit the Main Analysis Job**

Submit the main PBS job that processes all chromosome chunks:

```bash run_pipeline.sh submit```

This is the main stage of the workflow.

It will:

- process your assigned chromosomes
- run chunk by chunk
- skip already completed chunks
- write logs, summaries, checkpoints, and outputs into the group_runs directory

**Important:**

After submitting this stage, wait for the script/job to finish before running the merge step.
Do not run the merge step immediately after submission.
The runtime will depend on:
- number of BAM files
- chromosome size
- chunk size
- cluster load

**3. Check Progress**

To monitor the run, use:

```bash run_pipeline.sh progress```

This generates a progress report in your group output directory.

The report helps you check:

- how many chunks are complete
- which chunks are still missing
- whether a chromosome has finished
- whether the run is ready for merging

You can run the progress check as often as needed while the main job is running.

**4. Merge Final Outputs**

Only run this step after the main chunk-processing job has completed:

```bash run_pipeline.sh merge```

This merges all filtered chunk-level VCFs into final chromosome-level VCF files.

## Available Commands
```bash run_pipeline.sh <command>```

**Commands**

- setup	- Submit 01_make_regions.sh to create regions.tsv
- submit	- Submit 02_run_chunks.sh for parallel variant calling
- progress- Submit 04_check_progress.sh to generate a progress report

- help	- Show usage information


## Checking Whether the Job Is Still Running

You can check the queue using:

qstat -u $USER

If the submitted job is still listed, it is either running or waiting in the queue.

You can also inspect the pipeline output folders, especially:

- master_logs/
- logs/
- summaries/
- checkpoints/
- chunks/

## Resuming After Failure
Re-running the pipeline

In most cases, you can simply re-submit the main job:

```bash run_pipeline.sh submit```

Completed chunks will be detected and skipped automatically.

If some chunks failed midway

***Option A: Manual cleanup**

This is the safer option.

Check the logs in:

logs/chr*/
summaries/chr*/

Delete only the incomplete files for the failed region(s)

Re-submit the job

**Option B: Automatic cleanup of incomplete files**

```qsub -v AUTO_CLEAN_INCOMPLETE=1 02_run_chunks.sh```


## Configuration

Main settings are stored in pipeline.config.

Each processed chunk generates the following files:

```
File	Description
HCC.<chr>.<start>_<end>.bcf	Raw multi-sample BCF produced after mpileup and call
HCC.<chr>.<start>_<end>.annotated.bcf	BCF annotated with dbSNP rsIDs
HCC.<chr>.<start>_<end>.filtered.vcf.gz	Final compressed VCF after QUAL >= 20 filtering
```

Output Structure

Results are written into a chromosome-group-specific directory such as:

- group_runs/chr1_chr14_chr21/

This directory contains:

- regions.tsv — list of genomic chunks

- master_logs/ — overall run logs

- chunks/ — per-region outputs

- logs/ — detailed per-region logs

- checkpoints/ — completion stamp files
 
- summaries/ — timing and status files per region

- final/ — merged final chromosome VCFs

## Notes

This workflow is intended for chromosome-group-based processing in shared HPC environments.

Each chromosome is processed in chunks, allowing efficient restart and recovery.

Haploid calling is handled separately for chromosomes where diploid assumptions do not apply, such as chrM and chrY.
